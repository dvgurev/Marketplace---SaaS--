# Services

Каталог с описаниями независимых веб‑сервисов, доступных на маркетплейсе.

Каждый сервис представляет собой отдельное приложение, развёртываемое в
Docker-контейнере как тенант. В этой папке находятся шаблоны конфигураций, Dockerfile’ы,
скрипты для сборки образов, а также дополнительные метаданные (название, описание,
тарифы).

## Структура (пример)

```
services/
├── kanban-board/          # пример сервиса "Канбан-доска"
│   ├── Dockerfile
│   ├── config.yml         # описание параметров сервиса для оркестратора
│   └── README.md          # инструкция по сборке и развертыванию
├── crm/                   # CRM-сервис
└── ai-tool/               # AI-инструмент
```

## Добавление нового сервиса

1. Создать новую папку с уникальным идентификатором.
2. Определить Dockerfile и необходимые зависимости.
3. Заполнить метаданные (название, тарифы, ограничения).
4. Зарегистрировать сервис в API админки (через миграции или интерфейс).
