# Shared

Общие библиотеки, утилиты и модели данных, используемые как клиентом, так и
сервером (или другими проектами).

## Содержание (пример)

```
shared/
├── models/           # определения типов/интерфейсов (User, Subscription и т.д.)
├── utils/            # вспомогательные функции (валидация, форматирование)
├── constants/        # общие константы проекта
└── README.md
```

Цель – избегать дублирования кода, например при валидации payload’ов на фронте и
бэке.

## Использование

Добавить зависимость на `shared` модуль из `client` или `server` (npm link, go mod
replace и т.п.), импортировать необходимые пакеты.
