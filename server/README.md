# Server

Бэкенд/оркестратор платформы маркетплейса.

## Описание

Здесь реализована логика управления пользователями, подписками, интеграции с платежной
системой, API для фронтенда, а также механизмы создания и контроля Docker-контейнеров для
тенантов. Включает сервисы авторизации, биллинга, административный REST-интерфейс и т.д.

## Структура (пример)

```
server/
├── cmd/               # точка входа (main.go или подобное)
├── config/            # файлы конфигурации и загрузка
├── internal/          # приватные пакеты (auth, billing, tenant, etc.)
├── pkg/               # публичные библиотеки, используемые другими модулями
├── migrations/        # миграции для БД
├── scripts/           # утильные скрипты (резервное копирование, миграции)
├── Dockerfile
├── go.mod / package.json  # зависимости
└── README.md
```

## Запуск (локально)

```
cd server
# настроить переменные окружения, например скопировать .env.example в .env
# основной параметр DATABASE_URL должен указывать на Postgres (или sqlite при
# разработке с небольшими изменениями schemas)
npm install
npm run dev            # в режиме разработки
npm run build && npm start  # production
```

### Миграции

Применяйте файл `migrations/001_initial.sql` к вашей БД вручную или через `prisma`:

```bash
# если используется PostgreSQL и DATABASE_URL настроен
npx prisma migrate deploy
# (или) npx prisma db push --preview-feature
```

Для локального простого прототипа можно временно изменить схему Prisma на
`provider = "sqlite"` и задать `DATABASE_URL="file:./dev.db"`.
